# VMOS Architecture Hardening

## Goal
Turn the Phase 0-8 scaffolding into contract-visible, executable VMOS behavior without claiming incomplete Linux-personality or substrate work as evidence.

## Accepted Scope
Fix the concrete review findings around process/thread semantic records, portable snapshot restore, validator lookup, clone/wait execution state, signal delivery, page fault/COW plumbing, network backend integration, epoll/futex/file-lock/rlimit/seccomp/clock behavior, and validation gates. Work in small reviewable rounds; after each implementation round, run targeted validation, review the diff, fix discovered issues, and commit.

## Current Plan
Continue in small executable rounds. Round 2 and the first two Round 3 boundaries are complete and committed. The next Round 3 sub-round should either wire the packet pump/backend into the runtime network plane without overclaiming full TCP/IP completion, continue inside the process/signal execution boundary with the non-`vfork` child execution queue required before COW fork can be claimed, or move to another bounded subsystem gap such as Futex PI.

Validation: `cargo fmt --all --check`, `git diff --check`, `cargo check -p kernel --target x86_64-unknown-none`, `cargo test -p linux_syscall`, and a syscall-only `madvise_fork_advice_smoke` runner with VMOS execution trace evidence.

## Risks
Bounded dynamic `PT_INTERP` exec now maps a single interpreter and biased main image and supports VFS `pread64`, private file-backed `mmap`, bounded dynamic VFS `MAP_SHARED`, page-level `msync` writeback, bounded shared-file `MADV_REMOVE`, VMOS-visible `/proc/self/exe` readlink identity, non-stateful `madvise` hints, anonymous-page `MADV_DONTNEED` zero-discard/refault, bounded private file-backed `MADV_DONTNEED` snapshot reload, bounded `MADV_POPULATE_READ`/`MADV_POPULATE_WRITE` prefaulting, bounded anonymous `MADV_FREE` eager reclaim, bounded `MADV_DONTFORK`/`MADV_DOFORK`/`MADV_WIPEONFORK`/`MADV_KEEPONFORK` behavior for the current independent-fork path, `MAP_FIXED_NOREPLACE`, bounded in-place `mremap` same-size/shrink/tail-grow behavior, `MREMAP_MAYMOVE` auto-move when growth cannot happen in place, bounded `MREMAP_FIXED` relocation, and bounded anonymous `MREMAP_DONTUNMAP`; the initial Linux user loader has simple dynamic glibc smoke evidence when matching resource files are supplied. This is still only as useful as the resource VFS contents supplied at build time. The larger requested subsystem list is intentionally broad. The current fork/clone execution model is still child-runs-first with a suspended parent, not a general multi-runnable scheduler. `vfork` and shared-VM clone still share the active address-space representation; independent-VM `fork`, supported non-`CLONE_VM` clone, and `clone3` for the same bounded flag subset now use shared read-only COW mappings and child write-fault copy, but this is still not thread clone or a general scheduler COW implementation. Hidden parent fd-table references are used only while a suspended parent exists; arbitrary scheduler interleavings still need explicit cleanup paths before this can be generalized. Signal handler delivery is bounded to syscall-return checkpoints and a VMOS-private restorer frame; alternate stacks, `SS_AUTODISARM`, ABI-shaped `SA_SIGINFO` ucontext data, ucontext-backed restore for supported return registers, bounded RCX/R11 greg restore for the Linux-visible syscall-clobbered values, and bounded wait-like `SA_RESTART` now exist, but `rt_sigreturn` still uses VMOS private frame metadata for validation/FS-base/altstack state and does not implement full kernel sigcontext/FPU restore fidelity, pre-`syscall` RCX/R11 recovery, or Linux restart-block/timeout adjustment.
